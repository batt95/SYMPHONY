Getting Started

Here is a sketch outline of how to get started with SYMPHONY.

  • First, if you haven't already done so, unpack the distribution using the tar command.
  • You must obtain and install the Parallel Virtual Machine (PVM) software, available for free from Oak Ridge National Laboratories. See Section 1.6 for more notes on using PVM.
  • Edit the various path variables in the Makefile to match where you installed the distribution and where the PVM and LP libraries and header files reside for each architecture on your network. Other architecture-dependent variables should also be set as required.
  • Type ``make all'' in the SYMPHONY root directory. This should make all of the executables for the sample application, a vehicle routing problem solver.
  • Make sure there are links from your $PVM_ROOT/bin/$PVM_ARCH/ directory to each of the executables in the User/bin.$REV directory. This is required by PVM.
  • Start the PVM daemon by typing "pvm" on the command line.
  • Run the sample application by typing ``master User/par/sample.par''.
This should result in the solution of a small vehicle routing problem. Check out http://branchandcut.org/VRP/ if you want to solve more vehicle routing problems. Once you have accomplished this much, you are well on your way to having an application of your own. Don't be daunted by the seemingly endless list of user function that you are about to encounter. Most of them are optional or have default options. If you get lost, consult the source code for the sample application to see how it's done.
