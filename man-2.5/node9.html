Working with PVM

  To compile an application, it is necessary to install PVM. Currently, this is the case whether or not you plan to run anything in parallel. The current version of PVM can be obtained at http://www.ccs.ornl.gov/pvm/. It should compile and install without problems. You will have to make a few modifications to your .cshrc file, such as defining the PVM_ROOT environment variable, but this is all explained clearly in the PVM documentation. Note that all executables (or at least a link to them) must reside in the $PVM_ROOT/bin/$PVM_ARCH directory in order for parallel processes to be spawned correctly. The environment variable PVM_ARCH is set in your .cshrc file and contains a string representing the current architecture type. To run a parallel application, you must first start up the daemon on each of the machines you plan to use in the computation. How to do this is also explained in the PVM documentation.
